# Authentication troubleshooting boundary

The frontend is wired to the immutable cloud backend and displays its exact error response.

## Frontend-controlled behavior

- Request formatting
- Backend URL and API prefix
- Cookie creation
- Role redirect
- OTP countdown and resend UI
- Error presentation

## Backend-controlled behavior

- Whether the supplied Admin password matches the stored Neon hash
- Whether the account is active
- OTP insertion into `peripheralstalk.email_otps`
- Brevo SMTP delivery
- OTP verification
- User insertion into Neon
- JWT generation

Examples:

- `Invalid credentials` means FastAPI rejected the identifier/password.
- `Inactive account` means FastAPI found the user but the Neon row has `is_active = false`.
- `Failed to send OTP email` means FastAPI inserted or attempted to insert the OTP, but its Brevo SMTP call failed.
- `Registration failed` is generated by the immutable backend when its Neon transaction is rolled back.

These backend responses cannot be converted into successful authentication by frontend code without changing the backend or its deployment secrets.
